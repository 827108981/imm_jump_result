# 免疫跳值排查反馈报告生成工具 V2.0.5

正式交付只有两个入口：

- 桌面端：一线填写、RTS/GTS 审核、ZIP 导入导出，并可生成手机扫码同步任务。
- 手机扫码端：与当前电脑实时同步填写和照片，支持断点续传。
- 手机离线端：`手机离线采集.html`，单文件、离线填写、浏览器本机自动缓存。

## 启动开发版

```powershell
python -m pip install -r requirements.txt
python app.py
```

程序会自动打开浏览器。开发时如需固定端口，可先设置 `$env:JUMP_CHECK_PORT=5000`，再运行 `python app.py`；一线填写页为 `/`，RTS/GTS 审核页为 `/rts`。

标准模板固定为 `resources/M1武汉市第八医院-双大客户仪器故障反馈和排查表-20250412.xlsx`。该文件是开发运行、离线页生成和桌面端打包的必需资产。

## 业务规则

- 基础信息的设备型号仅支持 `CL-6000i`、`CL-8000i`；跳值项目为 Excel 中的下拉选项。
- `CL-6000i`：39 项。
- `CL-8000i`：普通项目 38 项，不显示清洗池流量；指定项目额外显示 7 项超声针排查，共 45 项。
- 每个照片位置最多 7 张；单张照片最大 5MB；ZIP 最大 500MB。
- 每个排查项展示 Excel 中对应的参考图片和实测记录示例，示例不自动写入实测记录。
- 手机端只导入 RTS/GTS 返回 ZIP；返回包包含原始一线数据，导入后可继续修改、补充并重新导出正式 ZIP。

## 打包发布

```powershell
python build_release.py
```

发布物生成在 `release/`：桌面端完整文件夹及 ZIP、单文件 `手机离线采集.html` 和使用说明。

## 手机端使用

将单文件发送到手机后，使用 Safari、Chrome、Edge 或系统浏览器打开。不要在微信内置浏览器中长时间填写。填写内容和照片保存在浏览器 IndexedDB；切换应用、锁屏或退出页面后再次打开同一浏览器仍可继续填写。
