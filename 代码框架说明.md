# 代码框架说明 V2.0

## 交付闭环

```mermaid
flowchart LR
  A[桌面端或手机离线端填写] --> B[一线正式 ZIP]
  B --> C[RTS/GTS 审核]
  C --> D[RTS/GTS 返回 ZIP]
  D --> E[桌面端或手机端直接导入]
  E --> F[补充后重新导出正式 ZIP]
```

## 关键契约

- 正式包格式：`frontline-report-v2`。
- 一线 ZIP 含 `manifest.json`、`report_data.json`、`report.html` 和 `images/`。
- RTS/GTS 返回 ZIP 含 `rts_review_data.json`，其中必须包含 `source_data` 和照片文件，手机与电脑据此恢复原始填写内容。
- `mobile_offline.html` 必须保持单文件，不得依赖 Flask、外网或相邻目录资源。
- 修改 Excel 规则或参考图片后，运行 `python build_offline_html.py`，再运行 `python build_release.py`。
